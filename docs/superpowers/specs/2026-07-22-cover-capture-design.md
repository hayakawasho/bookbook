# Cover capture design

## Goal

書影がない外部書誌の登録時に、欠けている書影と撮影操作を直接結び付ける。撮影は任意の補助操作とし、主操作の「新規追加」は現在の位置と強さを維持する。

## UI

- 書影がなく、撮影ハンドラーが渡されている場合だけ、103×145px の書影枠全体を撮影ボタンにする。
- 枠内には Material Symbols の filled `photo_camera` と「表紙を撮影」を縦に配置する。
- アイコンは既存の `ui/icon` と同じインライン SVG コンポーネントとして追加し、外部フォントや依存パッケージは増やさない。
- 撮影後は同じ枠を選択画像のプレビューに置き換える。書誌情報列には撮影ボタンを置かない。
- 「新規追加」はボトムシート最下部の主要ボタンとして維持する。

## Interaction and accessibility

- 画像枠のボタンから、非表示の `input[type=file]` を開く。
- 操作のアクセシブルネームは「表紙を撮影」の1つだけにする。ファイル入力は重複する操作として読み上げさせない。
- `accept="image/*"` と `capture="environment"`、選択後のリサイズ・プレビュー・登録時アップロードの既存フローは変えない。
- 書影が元からある場合は撮影操作を表示しない。

## Verification

- 統合テストで、書影なしの場合に「表紙を撮影」ボタンが1つ表示され、その操作からファイル選択、プレビュー、登録時アップロードまで到達することを確認する。
- 書影ありの場合に撮影ボタンが表示されないことを維持する。
- lint、Webビルド、全テストを実行し、実ブラウザで配置を確認する。

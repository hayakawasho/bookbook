## PR レビュー

`$ARGUMENTS` に PR 番号または差分を受け取り、以下の観点でレビューを行う。

### レビュー観点

1. **設計**: 責務の分離・抽象化のレベルは適切か
2. **コーディング規約**: プロジェクトの規約に準拠しているか
3. **型安全性**: 型エラーや unsafe なキャストがないか
4. **テスト**: 変更に対するテストが十分か
5. **パフォーマンス**: 明らかなボトルネックがないか
6. **セキュリティ**: 脆弱性の懸念がないか

### 出力形式

- 問題点は重要度 (🔴 must / 🟡 should / 🟢 nit) とともに列挙する
- 良い点も必ず記載する
